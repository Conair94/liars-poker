# Liars Poker Paper ‚Äî Project Bootstrap

## Overview

**Title:** "Strategic Analysis of Card-Based Liar's Poker: Combinatorial Foundations and Nash Equilibrium Approximation"
**Author:** Connor M. Lockhart (University of Maryland)
**Main file:** `Liars-poker.tex`

This paper analyzes a card-based variant of Liar's Poker played with a standard 52-card deck. Players bid on the quality of the combined pool of all players' hidden hands using standard poker hand rankings. The paper provides:
1. Exact/high-precision hand-type probabilities for pool sizes n=5..25
2. Conditional probabilities given private hand composition (6 conditions)
3. Framework for a future Regularized Nash Dynamics (RND) agent ‚Äî see `AGENT_DESIGN.md`

## Data Conventions

### Card Encoding
- Card index = rank √ó 4 + suit
- Rank: 0=2, 1=3, ..., 12=A
- Suit: 0=C, 1=D, 2=H, 3=S

### Hand Type Order (0..9, ascending strength)
```
0=High Card, 1=Pair, 2=Two Pair, 3=Three of a Kind,
4=Straight,  5=Flush, 6=Full House, 7=Four of a Kind,
8=Straight Flush, 9=Royal Flush
```

### Pool Sizes
- n=5..25 (total cards in the pool across all players' hands)
- n=5,6,7: exact combinatorial counts (no simulation needed)
- n‚â•8: Monte Carlo with 3,000,000 samples per n, seed=42

### Conditional Scenarios (6 conditions)
| Key | Description |
|-----|-------------|
| `pair` | 2 private cards of same rank |
| `adjacent` | 2 private cards with consecutive ranks |
| `suited` | 2 private cards of same suit |
| `3suited` | 3 private cards of same suit |
| `3range` | 3 private cards with max_rank ‚àí min_rank ‚â§ 4 |
| `trips` | 3 private cards of same rank |

## Script Specifications (for regeneration from scratch)

### poker_math_exact.py

Core library imported by all other scripts.

#### Card encoding
```
card_index = rank * 4 + suit
rank: 0=2, 1=3, ..., 12=A
suit: 0=C, 1=D, 2=H, 3=S
```

#### Hand type constants
```python
HIGH_CARD=0, PAIR=1, TWO_PAIR=2, THREE_OF_A_KIND=3, STRAIGHT=4,
FLUSH=5, FULL_HOUSE=6, FOUR_OF_A_KIND=7, STRAIGHT_FLUSH=8, ROYAL_FLUSH=9
```

#### `_evaluate(card_indices)` ‚Üí int (0..9)
Best 5-card hand from n card indices. SF/RF checked first, then 4K, FH, Flush, Straight, 3K, 2P, Pair, HC. Ace-low straight (A-2-3-4-5) counts as 5-high. Flush: any suit with 5+ cards qualifies.

#### `_evaluate_ranked(card_indices)` ‚Üí (int, int)
Returns `(hand_type, primary_rank)`:
- High Card: `max(ranks)`
- Pair: rank of the pair
- Two Pair: rank of the HIGHER pair only (secondary pair ignored by spec)
- Three of a Kind: rank of the trips
- Straight: high card of the best 5-card straight (Ace-low straight ‚Üí 3 = "5-high")
- Flush: highest card in the best 5-card flush
- Full House: rank of the trips only (pair rank ignored by spec)
- Four of a Kind: rank of the quads
- Straight Flush: high card of the SF sequence
- Royal Flush: always 12

## Important Design Notes

### Wu & Wu "contains" vs. "best-hand-equals"
A hand with 4 Aces still contains a Full House pattern; a Royal Flush still contains a Straight and Flush. Wu & Wu counts are reference data only ‚Äî we report P(best 5-card hand = T).

### Flush example hands
P(best hand is an X-high flush) is NOT linear in X. Low-card flushes are vanishingly rare. Always use A-high or K-high flush examples. Canonical: A‚ôÝ J‚ôÝ 9‚ôÝ 6‚ôÝ 3‚ôÝ.

### Threshold sub-type granularity
Threshold tables report the rank *within* a hand type (e.g. "Full House A" vs "Full House J"). Uses `_evaluate_ranked()` + rank-level MC. Per spec: Two Pair uses higher pair only, Full House uses trips only.

### Monte Carlo
- Blind: 3,000,000 samples per n, seed=42
- Conditional: 1,000,000 samples total, seed=42
- All MC output is cached to JSON; rerun loads cache unless stale
