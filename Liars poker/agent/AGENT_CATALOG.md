# Agent Catalog

All agent implementations for card-based Liar's Poker. Every agent is a
distinct named class in `agent/web/backend/agents.py` (Python) and mirrored
inline in `docs/index.html` (JavaScript).

**Rule**: Never silently repurpose an existing agent for a new game mode by
adding conditionals. Instead, add a new named class here so the catalog stays
accurate and old behaviour is preserved.

---

## Standard Rules (52-card deck, count-up or count-down)

### `RandomAgent`
- **Strategy**: Uniform random over all legal actions.
- **Data**: None.
- **Modes**: All modes (standard, exact, five-kings, high-hand).
- **Use for**: Baseline comparison; sanity testing.

### `BlindBaselineAgent`
- **Strategy**: 50% marginal threshold â€” bids the highest hand where
  P(pool â‰Ą bid | total cards = n) â‰Ą 0.5. Calls when standing bid exceeds
  the threshold.
- **Data**: `hand_rank_probs_matrix.json` (marginal MC, 3M samples, 52-card deck).
            Falls back to `blind_equilibrium.json` for n < 5.
- **Modes**: Standard rules only (count-up or count-down â€” same probabilities).
- **Note**: Ignores private cards entirely; good lower bound on strategic play.

### `ConditionalAgent`
- **Strategy**: Same threshold logic as BlindBaseline but adjusts the threshold
  upward when the private hand matches a strong condition (pair, trips, suited,
  etc.), using condition-specific conditional probability tables.
- **Data**: `extended_conditional_probs_ranked.json` (1M samples per condition,
            74 conditions Ă— 21 pool sizes).
            Falls back to `BlindBaselineAgent` for n < 5 or n > 25.
- **Modes**: Standard rules only.
- **Note**: Best general-purpose agent for standard rules.

## Exact Hand Rules (52-card deck, bid must have exact 5-card subset in pool)

### `ExactRulesBlindAgent`
- **Strategy**: Same 50% threshold logic as BlindBaseline, but uses
  P(pool contains exact 5-card subset == bid | n) instead of
  P(pool_best â‰Ą bid | n).  In exact mode high bids are much harder to
  justify (e.g. "Pair A" with pool [A,A,A,2,3] fails because no 5-card
  subset is *exactly* a pair of aces).
- **Data**: `exact_rules_probs.json` (generated by `compute_exact_rules_probs.py`,
            default 10k samples; run with `--samples 100000` for production).
- **Modes**: Exact Hand Rules mode only.
- **Availability**: Requires `exact_rules_probs.json` to exist; falls back to
  `BlindBaselineAgent` (standard probabilities) if missing.

---

## Five-Kings Mode (53-card deck, 5K Kings > SF)

### `FiveKingsBlindAgent`
- **Strategy**: Same 50% marginal threshold as BlindBaseline, calibrated for
  the 53-card deck.  Bid index 110 = Five of a Kind Kings is included in the
  probability table.
- **Data**: `five_kings_probs.json` (generated by `compute_five_kings_probs.py`,
            default 1M samples).
- **Modes**: Five-Kings easter egg mode only.
- **Availability**: Requires `five_kings_probs.json` to exist; falls back to
  `BlindBaselineAgent` (standard 52-card probabilities) if missing.

## Adding a New Agent â€” Checklist

1. Add the class to `agent/web/backend/agents.py` with a clear docstring naming
   what data it uses and what mode it targets.
2. Mirror the strategy in `docs/index.html` (JS) under `// 5. AGENTS`.
3. Add it to the setup dropdown in `renderSetup()` (JS) and the route handler
   in `agent/web/backend/app.py`.
4. Add an entry to this catalog with: strategy, data file, mode, fallback.
5. If it requires new MC data: add a `compute_*.py` script under `agent/data/`
   and add the file to the Data Files Summary table above.
